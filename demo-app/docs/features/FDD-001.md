# Feature: Create and list notes

**ID:** FDD-001  
**Domain:** Notes  
**Actor:** Sam (Casual Note-Taker)  
**Status:** DRAFT

## Feature Statement

Create a note with title and body and list all notes ordered by creation date (newest first).

## Business Value

Users can capture quick ideas and see them in a list without leaving the app.

## Screens

| Screen ID | Screen Name        | Job Story |
|-----------|--------------------|-----------|
| S01       | Note List Screen   | JS-002    |
| S02       | Create Note Screen | JS-001    |

## Backend Endpoints

- `POST /api/v1/notes` — create note (request: title, body; response: 201 with created note)
- `GET /api/v1/notes` — list notes, newest first (response: 200 with array of notes)

## Acceptance Criteria

1. A user can submit a title and body; the system creates a note and returns it with an id and createdAt.
2. A user can request the list of notes; the system returns all notes ordered by createdAt descending.
3. Title and body are required; empty title or body is rejected with 400.
